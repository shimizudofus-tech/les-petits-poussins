## 5. Captures d'écran requises

Google Play exige au minimum **2 captures** (recommandé : 4-8) au format :
- **Téléphone** : 16:9 ou 9:16, entre 320px et 3840px de côté, JPEG ou PNG 24 bits
- Taille recommandée : **1080 × 1920** (portrait)

### Captures à préparer (les fichiers seront dans `store-assets/`)
1. **Écran d'accueil** — ferme + animal (tamagotchi)
2. **Choix du niveau** — liste maternelle + primaire
3. **Exercice maternelle** — coloriage ou lettres
4. **Mini-jeux** — hub des 5 jeux
5. **Tracé au doigt** — lettre guidée
6. **Ferme** — exploration avec animaux
7. **Espace parent** — temps d'écran / stats

### Icône haute résolution
- **512 × 512 px**, PNG 32 bits, pas de transparence
- Déjà existante : `public/pwa-512x512.png`

### Bannière (Feature graphic)
- **1024 × 500 px**, JPEG ou PNG 24 bits
- À créer (illustration ferme + logo + "Jeu éducatif 3-11 ans")

## 7. Abonnement : essai gratuit + offre annuelle (config externe)

Aucun code à changer — tout se passe dans les consoles. Le code lit déjà l'offre RevenueCat.

### Play Console → Monétisation → Abonnements
1. **premium_monthly** (déjà prévu) : 1,99 €/mois
   - Ajouter une **offre** → **Essai gratuit 7 jours** (phase d'essai, prix 0, durée P7D)
2. Créer **premium_annual** : ex. 14,99 €/an (≈ 2 mois offerts)
   - Même entitlement `premium`

### RevenueCat → Offerings → "default"
- Ajouter 2 packages : `$rc_monthly` → premium_monthly, `$rc_annual` → premium_annual
- Lier les deux à l'entitlement `premium`

### Côté app (à faire quand le billing sera activé)
- Le paywall (`subscribe()` dans GameContext) prend `availablePackages[0]`.
  Pour proposer le choix mensuel/annuel, afficher les 2 packages de l'offering
  dans une UI de sélection avant l'achat. (Le reste — validation, essai, annulation — est géré par RevenueCat.)
